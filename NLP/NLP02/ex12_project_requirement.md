# Project: 챗봇 만들기

## 1. 프로젝트 개요
트랜스포머(Transformer) 모델을 활용하여 한국어 챗봇을 구현하고, 번역기 성능 측정법(BLEU Score)을 적용하여 모델을 평가합니다.

## 2. 세부 수행 단계

### Step 1. 데이터 다운로드
* **데이터셋**: [songys/Chatbot_data](https://github.com/songys/Chatbot_data) (`ChatbotData.csv`)
* **요구사항**: `pandas`를 사용해 데이터를 읽고 질문(`questions`)과 답변(`answers`)을 각각의 변수에 저장합니다.

### Step 2. 데이터 정제
* **함수**: `preprocess_sentence()` 구현
* **조건**:
    * 영문자는 소문자로 변환합니다.
    * 영문, 한글, 숫자, 주요 특수문자를 제외한 모든 문자를 정규식을 사용하여 제거합니다.

### Step 3. 데이터 토큰화
* **도구**: KoNLPy의 `Mecab.morphs` 사용
* **함수**: `build_corpus()` 구현
* **조건**:
    * 소스 문장, 타겟 문장을 입력으로 받아 preprocess_sentence() 함수로 정제하고 토큰화합니다.
    * 토큰화는 mecab.morphs 함수를 사용한다.
    * 토큰의 개수가 일정 길이 이상인 문장은 데이터에서 제외한다.
    * 소스 별, 타겟 별로 중복 문장을 제거하되, 질문-답변 쌍이 흐트러지지 않도록 유의한다.
    * questions 와 answers 를 각각 que_corpus , ans_corpus 에 토큰화하여 저장한다.

### Step 4. Augmentation (데이터 부풀리기)
* **기법**: Lexical Substitution (임베딩 유사도 기반 단어 교체)
* **도구**: [Kyubyong/wordvectors](https://github.com/Kyubyong/wordvectors)의 한국어 Word2Vec (`Korean (w)`, `ko.bin`) 사용
* **요구사항**:
    * 원본 질문 + 원본 답변
    * 증강 질문 + 원본 답변
    * 원본 질문 + 증강 답변
    * 위 세 가지를 조합하여 전체 데이터를 약 3배로 늘린다.

### Step 5. 데이터 벡터화
* **요구사항**:
    * 답변 데이터(`ans_corpus`)에 `<start>`, `<end>` 토큰을 추가한다.
    * 질문과 답변을 결합하여 단어 사전을 구축한다.
    * 데이터를 벡터화하여 `enc_train`, `dec_train`을 생성한다. (Embedding 층 공유 권장)

### Step 6. 훈련하기
* **모델**: 이전에 구현한 `Transformer` 모델 사용
* **요구사항**: 과적합을 피하기 위해 하이퍼파라미터 튜닝을 진행하고 다음 예문에 대한 답변을 생성한다.
    1. 지루하다, 놀러가고 싶어.
    2. 오늘 일찍 일어났더니 피곤하다.
    3. 간만에 여자친구랑 데이트 하기로 했어.
    4. 집에 있는다는 소리야.

### Step 7. 성능 측정하기
* **지표**: BLEU Score  
* **함수**: `calculate_bleu()` 적용
* **요구사항**: 육안 확인과 함께 정량적 지표인 BLEU Score를 통해 적절한 답변인지 평가한다.

## 3. 프로젝트 제출 루브릭
- **데이터 전처리**가 잘 이루어졌다 (노드10)
    - 데이터 정제, SentencePiece를 활용한 토큰화 및 데이터셋 구축 과정이 태스크에 맞게 진행되었다

- **Augmentation**을 적절히 수행하여 증강된 데이터셋을 확보하였다 (노드12)

- **과적합을 피하여 Transformer 모델이 안정적으로 훈련**되었다 (노드12)
    - 러닝커브! (train loss/acc, val loss/acc, test loss/acc) → 데이터셋도 나눠서 사용해야 한다

- **모델의 번역 / 생성 기능이 정상적으로 동작**한다 (노드10)

- 테스트 결과 **의미있는 수준의 생성문을 확보**하였으며, **Attention map의 시각화 결과**를 보여준다 (노드10)

- **평가지표를** 적용하여 생성문을 정량적으로 평가할 수 있다 (노드 12)

- **주어진 예문을 포함하여 챗봇에 던진 질문에 적절히 답하는 사례가 제출되었다.**

- **과적합을 피할 수 있는 하이퍼파라미터 셋이 적절히 제시되었다.**

- **챗봇 훈련데이터를 위한 전처리와 augmentation이 적절히 수행되어 3만개 가량의 훈련데이터셋이 구축되었다.**
